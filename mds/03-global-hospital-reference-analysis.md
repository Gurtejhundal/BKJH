# Global Hospital Reference Analysis

This file captures the UX pattern to borrow from strong hospital websites. Do not copy their visuals directly. Use their structural logic.

## What Strong Hospital Websites Do Well

### 1. They Prioritize Patient Actions

The strongest hospital homepages do not waste the hero section on abstract slogans. They quickly expose core actions:

- Request/book appointment
- Find doctor
- Find location
- Explore departments/specialties
- Contact support
- Emergency/help access

For this project, the hero must do the same.

## Hero Section Direction

### Hero Goal

Within 5 seconds, a visitor should know:

- Hospital name
- What the hospital provides
- How to call
- How to request appointment
- How to check OPD
- How to reach location

### Recommended Hero Layout

Desktop:

- Left: hospital name, short trust line, two CTA buttons
- Right: clean medical visual area or hospital image placeholder
- Below/overlapping: quick action cards

Mobile:

- Hospital name
- Trust line
- Emergency call button
- Appointment button
- Quick cards stacked
- Sticky bottom action bar

### Hero Content Structure

```text
Bibi Kaulan Ji Hospital
Trusted care for families, OPD consultation, emergency support, and essential medical services.

[Request Appointment] [Emergency Call]

Quick Access:
- OPD Timing
- Doctors
- Departments
- Ambulance
- Location
```

### Optional Punjabi Microcopy

```text
Emergency Care Available
ਐਮਰਜੈਂਸੀ ਸੇਵਾ ਉਪਲਬਧ
```

## Hero Should Not Do

Do not use:

- Huge meaningless headline like “Transforming Healthcare For Tomorrow”
- Fake global claims
- Overly cinematic animation
- Heavy stock imagery
- Text over busy photos
- Long paragraph above the fold

## Reference Pattern: Mayo Clinic

Useful lesson:

- Request appointment is direct.
- Find doctor and location are obvious.
- The site feels information-first, not decoration-first.

Apply this locally by putting appointment, doctor, OPD, and location access in the first screen.

## Reference Pattern: Cleveland Clinic

Useful lesson:

- Big navigation pathways: providers, locations, appointments/access.
- Clean institutional trust.

Apply this by using strong quick-action cards under the hero.

## Reference Pattern: Apollo Hospitals

Useful lesson:

- Indian patient UX requires visible doctor discovery, appointment booking, call now, specialties, emergency/contact.
- Doctor cards include specialization, experience, and appointment CTA.

Apply this to doctors and departments.

## Reference Pattern: Fortis Healthcare

Useful lesson:

- Doctor discovery, appointment pages, hospital/location trust and department routing are important for Indian hospital audiences.

Apply this through clean search/filter sections where practical.

## Final Hero Spec

### Desktop Hero

Components:

1. Top emergency strip or small emergency badge
2. Header
3. Hero headline
4. Short subtext
5. Primary CTA: Request Appointment
6. Emergency CTA: Emergency Call
7. Secondary CTA: Get Directions
8. Quick cards: OPD, Doctors, Departments, Ambulance
9. Visual area: hospital photo placeholder or abstract clean healthcare illustration

### Mobile Hero

Components:

1. Hospital name/logo
2. Short headline
3. Emergency Call button
4. Request Appointment button
5. Quick action grid
6. Sticky bottom bar

## Sample Final Hero Copy

```text
Bibi Kaulan Ji Hospital
Trusted hospital care for families, OPD consultation, emergency support, and essential medical services.

Book an appointment, check OPD timings, or call emergency support when care is needed urgently.
```

## Senior Execution Note

The hero must feel designed, not generated. Use real spacing, hierarchy, and restrained copy. The patient should never feel lost.
