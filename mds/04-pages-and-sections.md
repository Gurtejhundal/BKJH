# Pages and Sections

## Global Layout

Every public page uses:

- Header
- Page hero/banner
- Main content
- Relevant CTA block
- Contact strip
- Footer
- Mobile sticky action bar

## Page List

### 1. Home

Purpose: Give fast access to hospital services and main actions.

Sections:

1. Emergency top strip
2. Header/navigation
3. Hero with appointment/emergency CTAs
4. Quick action cards
5. About short intro
6. Departments preview
7. Doctors preview
8. OPD timing preview
9. Emergency and ambulance section
10. Services/facilities preview
11. Gallery preview
12. Contact/location section
13. Footer

### 2. About Hospital

Purpose: Explain the hospital simply and build trust.

Sections:

- Page intro
- Hospital story/overview
- Mission-style care statement
- Facilities summary
- Patient care values
- Contact CTA

Avoid fake claims unless verified by client.

### 3. Departments Listing

Purpose: Show all available departments.

Design:

- Department cards
- Icon/visual marker
- Short description
- “View Department” button

Each card links to a department detail page.

### 4. Department Detail Page

Purpose: Explain each department in more detail.

Sections:

- Department name
- Short overview
- Services/treatments list
- Related doctors
- OPD timings for that department
- Appointment CTA
- Emergency note if applicable

### 5. Doctors Page

Purpose: Help patients find available doctors.

Doctor card fields:

- Photo/fallback avatar
- Name
- Qualification
- Specialization
- Experience if provided
- OPD days
- OPD time
- Department
- Appointment button

Do not create separate doctor detail pages in v1 unless client explicitly asks later.

### 6. OPD Timing Page

Purpose: Make timings clear and searchable.

Sections:

- Timing table
- Filters by department/doctor if practical
- Notes about holidays/emergency
- Appointment CTA

Timing fields:

- Department
- Doctor
- Days
- Start time
- End time
- Room/location if applicable
- Notes

### 7. Appointment Page

Purpose: Collect appointment requests.

Form fields:

- Patient name
- Phone number
- Department
- Preferred doctor
- Preferred date
- Preferred time/session
- Message/symptoms short note
- Consent checkbox

After submission:

- Show success message
- Tell patient staff will contact them
- Do not promise confirmed appointment automatically

### 8. Emergency Service Page/Section

Purpose: Immediate emergency contact.

Sections:

- Emergency number
- Emergency availability
- Important patient instruction
- Location shortcut
- Ambulance shortcut

Use red carefully.

### 9. Ambulance Page/Section

Purpose: Let users call ambulance quickly.

Fields:

- Ambulance number
- Availability status
- Service area note
- Call button
- Location link

### 10. Gallery Page

Purpose: Show real hospital environment.

Style:

- Category-based gallery
- Categories: Hospital, Doctors, Facilities, Camps, Events
- Grid layout
- Lightbox optional

Do not upload random photos. Real hospital photos only.

### 11. Services Page

Purpose: List hospital services in simple language.

Examples:

- OPD consultation
- Emergency care
- Ambulance support
- General medicine
- Minor procedures
- Diagnostic support if available
- Pharmacy if available

Only include services confirmed by client.

### 12. Facilities Page

Purpose: Show hospital infrastructure.

Examples:

- Waiting area
- Emergency room
- Ambulance support
- Pharmacy
- Lab
- Ward/rooms
- Operation/procedure room

Only include what exists.

### 13. Contact / Location Page

Purpose: Help patients reach hospital.

Sections:

- Address
- Phone numbers
- Emergency number
- Ambulance number
- Google Map embed
- Direction button
- Working/OPD hours
- Contact form optional

## Mobile Sticky Bottom Bar

Always include:

- Call
- Appointment
- Location
- Emergency

## Navigation Structure

Desktop nav:

- Home
- About
- Departments
- Doctors
- OPD Timing
- Appointment
- Gallery
- Contact

Mobile nav:

- Use a clean slide/dropdown menu
- Keep emergency and appointment buttons visible

## Content Fallbacks

If client has not provided content:

- Use clean placeholders in admin/dev only
- Do not publish fake claims
- Do not publish fake doctor data
- Keep unpublished items hidden

## Page Quality Rules

Each page must have:

- Clear page title
- Short intro
- Main content
- Patient action
- Mobile readability
- No filler sections
