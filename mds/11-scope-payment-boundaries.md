# Scope and Payment Boundaries

## Agreed Context

The client is a family doctor/relative, so the final charged amount may be discounted. That does not mean unlimited work is included.

The professional boundary is simple:

- Build the agreed website well.
- Do not silently include advanced hospital software features.
- Do not start unpaid add-ons.

## Included Scope

Included in current website project:

- Home
- About
- Departments
- Department detail pages
- Doctors
- OPD timings
- Appointment request form
- Appointment admin status management
- Emergency service
- Ambulance information
- Gallery
- Services
- Facilities
- Contact/location
- Basic admin content control
- Mobile responsive design
- Basic SEO setup
- Deployment support

## Admin Included

Admin can manage:

- Doctors
- Departments
- OPD timings
- Gallery
- Services
- Facilities
- Appointment requests
- Emergency number
- Ambulance number
- Notices
- About content

## Not Included

These are separate paid features:

- Patient login
- Patient medical records
- Lab report upload/download
- Online payment
- SMS API
- WhatsApp API automation
- Doctor slot booking engine
- Queue token system
- Pharmacy/inventory
- Billing system
- Staff attendance
- Full hospital management software
- Mobile app
- Advanced analytics dashboard

## Suggested Payment Structure

For ₹30,000 final project amount:

- ₹10,000 advance to start
- ₹10,000 after first working demo
- ₹10,000 before final deployment/handover

Do not hand over admin credentials or final deployment before final payment.

## Revision Boundary

Included:

- Reasonable content corrections
- Minor design adjustments
- Phone/address/timing corrections
- Basic layout polishing

Not included:

- Changing entire design direction after approval
- Adding new modules
- Rebuilding appointment as full slot system
- Rewriting all content multiple times
- Adding new language system
- Adding payment/SMS/WhatsApp automation

## Client Content Responsibility

Client must provide:

- Correct hospital name
- Correct phone numbers
- Correct emergency/ambulance numbers
- Doctors list
- Departments list
- OPD timings
- Gallery photos
- About details
- Services/facilities list

Wrong content provided by client is not developer fault.

## Approval Milestones

### Milestone 1 — Design Approval

Client approves homepage and overall design direction.

### Milestone 2 — Content Approval

Client approves doctor, department, timing, and contact details.

### Milestone 3 — Final Approval

Client approves final website before launch.

## Client Message for Scope

```text
Sir, the website will include the agreed hospital pages, appointment request form, and basic admin panel to manage doctors, departments, OPD timings, gallery, services, and appointment requests.

Advanced features like SMS/WhatsApp automation, online payment, patient records, lab reports, or full booking slot system are separate modules and can be added later if required.
```

## Brutal Reality

Relative projects become messy because people assume relationship means unlimited changes. Keep the tone respectful, but keep the boundary firm.
