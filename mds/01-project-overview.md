# Project Overview

## Project Name

Bibi Kaulan Ji Hospital Website

## Service Provider

Gurtejbir Singh

## Project Type

Dynamic hospital website with admin-controlled content.

## Business Context

The hospital needs a professional online presence where patients can quickly find departments, doctors, OPD timings, emergency support, ambulance information, gallery/facilities, appointment request options, and location details.

The client is a family doctor/relative, so the commercial pricing is discounted. The scope must still remain professionally controlled.

## Primary Objective

Create a mobile-first, trustworthy hospital website that improves patient access to hospital information and reduces manual explanation over phone/WhatsApp.

## Secondary Objective

Give hospital staff a simple admin system to update key public content without developer involvement.

## Product Boundary

This is a **hospital website with admin panel**, not a full hospital management system.

### Included in v1

- Homepage
- About page
- Departments listing
- Department detail pages
- Doctors page
- OPD timing page
- Appointment request form
- Emergency service page/section
- Ambulance page/section
- Gallery
- Services
- Facilities
- Contact/location page
- Admin-controlled content
- Appointment request status management
- Mobile sticky call/action bar

### Not Included in v1

- Patient login
- Medical records
- Lab reports
- Payment gateway
- SMS/WhatsApp API automation
- Advanced doctor slot scheduling
- Queue token system
- Pharmacy stock
- Billing system
- Full hospital ERP/HMS

## Target Users

### Patients

Mostly mobile users who need fast access to:

- Phone number
- Appointment request
- OPD timing
- Doctor list
- Department information
- Emergency support
- Ambulance number
- Location

### Hospital Staff/Admin

Users who need to:

- Add/edit doctors
- Add/edit departments
- Update OPD timing
- View appointment requests
- Update emergency/ambulance information
- Upload gallery images
- Add notices/health camp updates

## Success Criteria

The project is successful when:

- The homepage explains the hospital clearly within 5 seconds.
- Mobile users can call, request appointment, open location, or emergency number without searching.
- The admin can update major content without developer help.
- The website loads fast on average Indian mobile networks.
- The design feels senior, polished, and non-template.
- The system avoids over-engineering.

## Recommended v1 Strategy

Build the clean version first. Use placeholders where final hospital photos are missing, but structure the layout so real photos can be dropped in later without redesigning the website.

## Risk Control

The biggest risk is not Django. The biggest risk is scope creep.

Do not allow the project to silently become hospital management software.
