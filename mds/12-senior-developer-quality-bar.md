# Senior Developer Quality Bar

Use this before showing anything to the client.

## The Website Must Not Look AI-Generated

Common AI-looking mistakes:

- Generic headline: “Your Health, Our Priority” with no local specificity
- Same repeated cards everywhere
- Random gradients
- Fake doctor stock images
- Too much centered text
- No real spacing system
- No mobile UX thinking
- Overuse of icons
- Empty sections only to make page longer
- Buzzwords without evidence

Avoid all of these.

## Senior-Level Visual Rules

### Spacing

- Use consistent vertical spacing.
- Do not squeeze sections together.
- Do not leave random empty gaps.
- Mobile spacing should be tighter but readable.

### Typography

- One clear H1 per page.
- H2 section headings should be concise.
- Body text should be readable, not tiny.
- Line-height must be comfortable.

### Cards

- All cards should have consistent padding.
- Icons should be same style.
- Button alignment should be consistent.
- Cards must not have random heights unless grid handles it cleanly.

### CTAs

Every major page should guide the patient to one of:

- Call
- Appointment
- Location
- OPD timing
- Doctor/department

### Mobile

Mobile is not an afterthought.

Check:

- Header fits
- Hero does not become huge
- Buttons are thumb-friendly
- Sticky bottom bar does not cover content
- Forms are easy to use
- Tables scroll or stack properly
- Phone links work

## Copywriting Rules

Use direct copy.

Good:

```text
Request an appointment and our staff will contact you for confirmation.
```

Bad:

```text
Experience revolutionary healthcare excellence with transformative medical solutions.
```

Good:

```text
Check OPD timings before visiting the hospital.
```

Bad:

```text
Seamlessly explore our integrated medical touchpoints.
```

## Hospital Trust Rules

Only use verified claims.

Do not write:

- Best hospital
- World-class
- No. 1
- Advanced technology
- 24/7 emergency
- Expert doctors

Unless client confirms and it is true.

Use safer wording:

- Trusted care
- Experienced doctors if confirmed
- Emergency support if confirmed
- OPD consultation
- Ambulance support if confirmed

## Page Review Checklist

Each page should pass:

- Is the main purpose clear?
- Is the main CTA visible?
- Does it work on mobile?
- Is there any fake/filler text?
- Is the layout consistent?
- Are images optimized?
- Are empty states handled?
- Are phone/map links correct?

## Homepage Above-the-Fold Checklist

Must show:

- Hospital name
- Short clear line
- Appointment button
- Emergency call button
- OPD/location access

If a patient cannot find the phone number quickly, the homepage has failed.

## Department Detail Quality

Each department page should include:

- What the department is for
- What services it provides
- Which doctors are related
- OPD timing
- Appointment CTA

Do not leave department pages thin and useless.

## Doctor Card Quality

Each doctor card should include:

- Name
- Qualification
- Specialization
- Department
- OPD timing
- Appointment button

If photo missing, use professional initials/avatar fallback.

## Appointment UX Quality

The form must be simple.

Do not ask unnecessary questions.

Required fields only:

- Name
- Phone
- Department
- Preferred date

Optional:

- Doctor
- Message
- Preferred time

## Gallery Quality

Use real photos. Bad photos damage trust.

Rules:

- Avoid blurry images
- Avoid private patient faces
- Use categories
- Compress images
- Add alt text

## Final Client Demo Rule

Do not show the client a half-broken flow. Show fewer things polished instead of many things unfinished.
