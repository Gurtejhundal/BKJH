# Content Requirements From Client

This file should be sent to/used with the client before final content entry.

## Basic Hospital Information

Required:

- Full hospital name
- Short hospital description
- Full address
- Google Maps link
- Main phone number
- Emergency phone number
- Ambulance phone number
- WhatsApp number if used
- Email address if any
- Opening/OPD hours

## Branding

Required if available:

- Logo
- Hospital board photo
- Any preferred color
- Doctor/hospital letterhead if available

If no logo exists, use temporary text-based logo.

## About Page Content

Ask client for:

- When the hospital started
- Founder/doctor name
- Main type of care provided
- Key services
- Facilities
- Patient care approach
- Any verified achievements/certifications

Do not invent awards, years, facilities, or certifications.

## Departments

For each department:

- Department name
- Short description
- Main services
- Related doctors
- OPD days/timing
- Emergency relevance if any

## Doctors

For each doctor:

- Full name
- Photo
- Qualification
- Specialization
- Experience
- Department
- OPD days
- OPD time
- Contact/appointment preference if any

## OPD Timing

Collect:

- Department
- Doctor
- Days
- Start time
- End time
- Room number if any
- Notes

Example format:

| Department | Doctor | Days | Time | Notes |
|---|---|---|---|---|
| General Medicine | Dr. Name | Mon-Sat | 10 AM - 2 PM | Appointment recommended |

## Appointment Form Configuration

Confirm:

- Should appointment requests go to admin panel only?
- Should email notification be sent?
- Should WhatsApp button also be shown?
- Who will call patients back?
- What message should appear after form submission?

Recommended success message:

```text
Your appointment request has been received. Hospital staff will contact you for confirmation.
```

## Emergency Details

Collect:

- Emergency number
- Emergency availability: 24/7 or limited
- Emergency services provided
- Any patient instruction

Do not write “24/7 emergency” unless confirmed.

## Ambulance Details

Collect:

- Ambulance number
- Availability
- Service area
- Charges if client wants to mention
- Contact person if any

## Gallery Photos

Ask for real photos:

- Hospital exterior
- Reception
- Waiting area
- Doctor rooms
- Emergency area
- Ambulance
- Facilities
- Events/camps

Photo rules:

- Avoid patient faces unless permission exists.
- Avoid sensitive medical scenes.
- Use clean, bright images.
- Compress before upload.

## Services

Collect confirmed service list only.

Possible examples:

- OPD consultation
- Emergency care
- Ambulance service
- General medicine
- Pediatrics
- Gynecology
- Orthopedics
- Diagnostics
- Pharmacy
- Minor procedures

Remove anything not confirmed.

## Facilities

Collect confirmed facility list only.

Possible examples:

- Reception
- Waiting area
- OPD rooms
- Emergency support
- Ambulance
- Pharmacy
- Lab
- Wards

## Punjabi Microcopy

Use only for important action labels.

Examples:

- ਐਮਰਜੈਂਸੀ ਸੇਵਾ ਉਪਲਬਧ
- ਮੁਲਾਕਾਤ ਲਈ ਬੇਨਤੀ ਕਰੋ
- ਸਥਾਨ ਵੇਖੋ

Do not translate the entire site in v1.

## Content Approval Rule

Before launch, send client a content preview and get approval for:

- Doctors
- Departments
- Timings
- Phone numbers
- Emergency/ambulance numbers
- Address

Wrong hospital contact info is not a small bug. It is a serious trust problem.
